05 NEO4J GRAPH MEMORY
Node Types
  • Person — name, relationship type, last interaction, importance score
  • Event — calendar event, date, attendees, outcome
  • Task — description, status, deadline, priority, owner
  • Topic — project/subject, linked persons and decisions
  • Decision — what was decided, when, context, who was involved
  • Location — place name, coordinates, visit frequency
  • Document — file/PDF, summary, key points
  • Reminder — trigger type (time or location), message, status
  • Commitment — promise made by/to someone, deadline
Relation Types
  • KNOWS — Person ↔ Person
  • ASSIGNED_TO — Task → Person
  • RELATED_TO — Topic ↔ Topic / Event / Person
  • DECIDED_IN — Decision → Event
  • WORKS_ON — Person → Topic
  • LOCATED_AT — Event / Person → Location
  • REFERENCES — Task / Decision → Document
  • COMMITTED_TO — Person → Task (promised)
  • FOLLOWS_UP — Task → Decision (action from decision)
Example Query — "Prepare me for my project meeting tomorrow"
1. MATCH (e:Event {title:"project meeting"}) → get attendees
2. For each person: MATCH (p)-[:COMMITTED_TO|DECIDED_IN]→ open tasks + past decisions
3. MATCH (t:Topic)-[:RELATED_TO]→(e) → get related projects + open issues
4. Build context string → send to Gemini: "Write a meeting briefing"
→ Result: Per-person briefing with past decisions, open tasks, and key context
06 LLM STRATEGY
Qwen2.5 7B Instruct — Router
  • Switch immediately from Coder → Instruct variant
  • Runs entirely local via Ollama on your 4050 RTX (6GB VRAM)
  • Role: parse intent + select tools + extract parameters via CoT
  • Never generates final user-facing responses
  • English prompts only — dramatically better tool selection
  • Why not 14B? Doesn't fit in 6GB VRAM, would spill to RAM, slow + unstable
Gemini Flash 2.0 — Finalizer
  • Receives: tool results + Neo4j context + conversation history
  • Role: reason, synthesize, generate final response
  • Handles all "hard thinking" — summaries, plans, briefings
  • Free tier sufficient for personal use
  • Long context: handles entire email threads, PDFs, meeting notes
  • Turkish response mode: one system prompt line when stable
Chain of Thought Flow
User: "Add a meeting tomorrow at 3pm with Ahmet about the project"
─────────────────────────────────────────────────────
Step 1 [Qwen — Intent]: "User wants to create a calendar event."
Step 2 [Qwen — Tool]: "Required tool: calendar_write. Also check contacts for Ahmet."
Step 3 [Qwen — Params]: {"title": "Project meeting", "date": "tomorrow", "time": "15:00", "attendee": "Ahmet"}
Step 4 [Execute]: calendar.create_event(...) → contacts.search("Ahmet") → event created
Step 5 [Gemini]: "Done. Added 'Project meeting with Ahmet' tomorrow at 15:00. Want me to send him a notification?"
Step 6 [Memory]: Neo4j updated — Event node linked to Person[Ahmet], Topic[project]

10 ENGLISH-FIRST STRATEGY
Why English-First
  • Qwen 7B Instruct performs significantly better on English prompts for tool calling
  • Gemini Flash reasoning quality is higher with English context
  • Eliminates MarianMT translation layer — a latency + failure point
  • Easier debugging: one language across intent → tools → memory → output
  • Tool descriptions, Neo4j nodes, system prompt all in one language
Turkish Activation Plan
  • Trigger: system runs stable for 2–3 weeks, all Phase 2 tools working
  • Implementation: add one line to Gemini system prompt: "Always respond in Turkish."
  • No architecture changes needed. Zero refactoring.
  • Internal processing (routing, memory, tools) stays English forever
  • User sees Turkish. Bantz thinks in English. Perfect.
